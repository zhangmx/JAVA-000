要实现一个业务网关，猜测应该有进有出，是业务的吗？还是通用类似nginx那种？

1. 首先，这应该是一个web server，启动后，提供对外服务。
2. 提供一些类似nginx配置文件那种定义的conf或者yaml。控制哪些入口对应哪些后端服务。
3. 造几个（先来一个）后端服务，假装接入到网关中。

### 验证

用户（client）请求网关端口，可以获取到后端服务返回的内容。
